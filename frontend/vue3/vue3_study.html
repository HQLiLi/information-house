一.使用create vue创建vue3项目 ​

1.创建vue3项目 ​

​ 与创建vue2项目不同，vue2的项目是使用vue cli进行创建，但vue3我们更推荐使用create-vue来创建，因为create-vue底层是使用了下一代的创建工具（vite），这为我们的开发提供了急速响应。

​ 而我们在创建项目时的指令也发生了一点变化,大致分为以下几步

  1. 确认node.js的版本(16以上即可)
  2. 执行 npm init vue@latest 指令 (第一次使用会要求安装create vue 的依赖文件)
  3. 根据步骤创建vue3项目
  4. 执行 npm install 指令
  5. 执行 npm run dev 指令 启动项目

2.熟悉vue3项目目录及关键文件 ​

​ 注意：在我们通过VsCode来启动vue3项目时，插件可能会弹出提示，因为我们在vue2中使用的插件时Vetur，而vue3使用的时Volar，所以我们需要禁用Vetur并安装Volar。

​ 我们从中挑出main,js来讲讲,这个比较关键。

js
main.js
------------------------------
import './assets/main.css'

// 从vue中解构出createApp
// new Vue() 创建一个应用实例=>createApp()
// 将创建实例进行了封装，保证了每个实例数据的独立封闭性
import { createApp } from 'vue'
import App from './App.vue'


// mount 设置挂载点 #app (id 为app 的盒子)
createApp(App).mount('#app')

二.组合式API ​

1.setup ​

​ 讲到了组合式API，那自然少不了setup。

​ 所谓组合式API，可以理解为是一系列函数，而setup则是入口。

​ setup类似我们的生命周期钩子，是一个函数，而且它的执行顺序在 beforeCreat( ) 之前。

​ 当然这也就意味着我们无法在setup中获取this，因为vue只有在Create之后才会将this指向vue实例，而setup在create之前，自然this就是undefind了。当然，我们也无需担心，因为this本身也是跟着所处环境变化而变化，我们在vue3中this基本上不用。

​ 我们在setup()中定义的属性和函数并不能直接在模板上使用，需要先retrun才行。

​ return之后使用就跟之前一样。

vue
<script>
export default{
  setup(){
    const msg = `red`
    const doMsg = () =>{
      console.log(msg)
    }
    
    return {
      msg,
      doMsg
    } 
  }
}
</script>

<template>
  <div>{{  msg }}</div>
  <button @click="doMsg">打印</button>
</template>

​ 但是如果我们的属性和函数过多，我们还得一 一return回去，这样子效率低且麻烦，因此我们可以使用setup的语法糖。

​ 而使用了语法糖后，我们代码就会变成这样:

vue
<script setup>
    const msg = `red`
    const doMsg = () =>{
      console.log(msg)
    }
</script>

<template>
  <div>{{  msg }}</div>
  <button @click="doMsg">打印</button>
</template>

​ 与先前的代码进行比较，光简洁不说，还十分地简便。

​ 当然，setup 中的属性和函数还是要 return，但毕竟你不return 有的是人return ，return的步骤已经交给了setup的语法糖了，我们可以看一下setup语法糖的底层原理：

2.reactive & ref ​

​

​ 在vue3中，默认的数据并不是响应式。这点也很好理解，因为我们的属性与方法是在setup()中写的，而vue开始将数据变成响应式数据要等到create()后，因此我们要通过reactive()和ref()来将数据变成响应式数据。

​ 其实在vue3中，我们如果不用reactive()和ref()来将数据变成响应式数据，也能将内容渲染到页面上。

vue
<script setup>
const obj={
  name:`red`,
  salary:1600
}
const addSalary = ()=>{
  obj.salary+=100
}
</script>

<template>
<div>{{ obj.name }}</div>
<div>{{ obj.salary }}</div>
<button @click="addSalary">转换</button>
</template>

​

​ 但是我们是无法修改它们，就像是通过v-bind绑定在input上的数据一样，单纯改变是无法影响到原数据。也正是因此，我们才需要通过reactive()和ref()来将数据变成响应式数据。

【Ⅰ】reactive（） ​

​ reactive（）的作用是将复杂类型的数据(即对象)变成响应式数据。

vue
<script setup>
    //1.从vue中导入reactive
import { reactive } from 'vue';
	//2.在reactive()中传入对象的初始值，并用变量接收
  const obj =  reactive({
    name:`red`,
    salary:1600
  })
const addSalary = ()=>{
  obj.salary+=100
}
</script>

<template>
<div>{{ obj.name }}</div>
<div>{{ obj.salary }}</div>
<button @click="addSalary">转换</button>
</template>

​ 通过reactive()，我们已经成功地能够将对象变为响应式数据，但reactive只能将复杂类型变为响应式数据，不能将简单类型变为响应式数据，所以这就需要ref登场了。

【Ⅱ】ref() ​

​ ref不仅能够将简单类型变为响应式数据，还能将复杂类型也变为响应式数据。

vue
<script setup>
    // 1. 导入ref
import { ref } from 'vue';
	// 2.1 将简单类型的初始值用ref()包住，并用变量接收 
const hero= ref(`red`)
	// 2.2 将复杂类型的初始值用ref()包住,并用变量接收 
const obj = ref({
  name:`大方老师`,
  salary:1600
})

const addSalary = () =>{
  obj.value.salary++
}

const changeHero = () =>{
  hero.value = `大桐老师`
}


</script>

<template>
  <div>{{ hero }}</div>
  <button @click="changeHero">转换</button>
  <div>{{ obj.name }}</div>
  <div>{{ obj.salary }}</div>
  <button @click="addSalary">增加</button>
</template>

​ 通过ref我们成功是实现了将简单数据变为响应式数据，但是我们可以明显的看出ref在获取响应式数据的值的时候好像不太向我们以往vue2中获取一样，关于这点，我会在将reactive和ref的区别的时候进行说明。

【Ⅲ】reactive和ref的区别 ​

​ 在前面我们可以看出reactive和ref都能将复杂数据变为响应式数据，但是这两者有着很明显的区别。

​ 首先，ref无论是把简单类型还是复杂类型变成响应式数据，底层都是调用了reactive实现了响应式，但因为reactive只能将复杂类型变为响应式，所以ref在调用reactive之前先进行了一次包装，将我们传入的值变为了复杂类型。我们可以通过控制台打印来查看。

​ 根据控制台的信息我们可以得知，我们传入ref的值，都放在了它包装成对象中的value中，这也就证明了为什么在上面的方法中要通过 变量.value.XX || 变量.value 来获取值。这里要总结一下：

  1. ref获取的响应数据想要在脚本(即script)中获取必须通过属性,value的方式
  2. ref获取的响应数据在模板(即template)中不用加上value，因为vue会自动帮我们取消包装，我们正常使用即可。

​ 既然展示了ref转换的数据在控制台中的样子，我们也顺便展示一下reactive转换的数据在控制台中的样子。

​ (打印了两次)从图上我们可以得知，通过reactive获取的数据就是我们平时获取的对象的形式，因此不需要加value。

​ 最后总结一下，虽然说reactive和ref都能将数据变为响应式数据，但我们更推荐使用ref，因为

  1. ref不仅将复杂数据变为响应式数据，连简单数据也能变为响应式数据
  2. 在初学阶段容易将两个搞混。

3.computed ​

​ vue3中的computed的思路与vue2的是大差不差的，都是对一个变量进行计算等操作返回一个新的值。但在vue3中它不在是作为配置项，而是作为一个函数，这也是与 组合式API这个思想有关，与这个API有关的都放在一起，而不是分散开来。

vue
<script setup>
    // 1. 导入computed
import { computed, ref } from 'vue';

const arr= ref([1,2,3,4,5,6,7])
	// 2. computed中放方法，并用一个变量接收返回值
const comArr = computed( () => arr.value.filter((item)=>item>3))
</script>

<template>
  <div>计算前:{{ arr }}</div>
  <div>计算后：{{ comArr  }}</div>
</template>

​ 上面展示的是computed最常用的方法，但学过vue2我们知道，computed不仅有get，也还有set。

vue
<script setup>
import { computed, ref } from 'vue';

const arr= ref(10)

const comArr = computed({
  get:()=>arr.value,  
  set:(num)=>arr.value+=num
})
</script>

<template>
  <div>原属性:{{ arr }}</div>
  <div>计算后：{{ comArr }}</div>
		//comArr= 的值会成为set()中的形参
  <button @click="comArr=3">转变</button>
</template>

​ 但是这种并不常用，除非特殊情况，我们尽量不要这么去做，因为这样会影响到源数据。

4.watch ​

vue3的watch和vue2的watch在思想上也没有什么区别，也像computed一样为了适应组合式API理念，从配置项变成了个函数，而在讲computed的时候提到了在computed中最好不要进行什么异步请求等操作，因为这些最好放到watch中。

【Ⅰ】watch监听简单类型属性 ​

vue
<script setup>
import { ref, watch } from 'vue';

const price = ref(1600)

const addPrice = () =>{
  price.value+=100
}
// watch(ref对象,回调函数)
watch(price,(newValue,oldeValue)=>{
  console.log(`老值：${oldeValue},新值:${newValue}`)
})
</script>

<template>
  <div>{{ price }}</div>
  <button @click="addPrice">转变</button>
</template>

​ 我们可以看出，watch在监听简单数据类型的时候，跟原来没什么两样，但是watch不能直接监听对象。这点我们放到下次说。、

【Ⅱ】watch监听复杂类型属性 ​

​ 前面也提到了watch无法直接监听复杂类型，即对象。这是因为对象是引用数据类型，如果我们只是修改对象中的一个值的话，它的地址没变，watch就认为这个对象没有发生改变，说白了，watch是将ref.value看成了一个整体，而这也被成为浅度监听。

​ 如果我们需要监听watch中对象的值的改变，就要使用到watch的深度监听。

vue
<script setup>
import { ref, watch } from 'vue';
    
const obj = ref({
  name:`red`,
  salary: 1600
})

const addSalary = () =>{
  obj.value.salary++
}

// watch深度监听，只需要在回调函数后面跟个对象{}，对象里加入如deep:true即可
// 同理，immediate(立即执行一次)也是如此
watch(obj,(newValue)=>{
  console.log(newValue)
},{
  deep:true
})
</script>

<template>
  <div>{{ obj.salary }}</div>
  <button @click="addSalary">转变</button>
</template>

【Ⅲ】watch监听复杂类型 ​

​

​ 在前面我们讲了watch能够通过设置deep参数来监听复杂类型属性，但是我们可以发现，通过deep监听的是整个对象，也就是说监听的是对象中的所有属性，如果我们要监听对象中的某个属性，则可以使用以下的方法。

vue
<script setup>
import { ref, watch } from 'vue';


const obj = ref({
  name:`red`,
  salary: 1600
})

const addSalary = () =>{
  obj.value.salary++
}
// 针对对象中的单个属性，进行监视
// watch(
// ()=>obj.value.XXX
// ()=>console.log(`xxx`)
//   )
watch(()=>obj.value.salary,(newValue)=>{
  console.log(newValue)
})
</script>

<template>
  <div>{{ obj.salary }}</div>
  <button @click="addSalary">转变</button>
</template>

【Ⅳ】watch监听多个数据 ​

​ watch不仅能够监听单个数据，还能监听多个数据。

vue
<script setup>
import {  ref, watch } from 'vue';

const hero = ref(`red`)
const price = ref(1600)
// watch([x,x,x..],回调函数)
watch([hero,price],(newValue,oldValue)=>{
  console.log(`新制：${newValue}`)
  console.log(`老值：${oldValue}`)
})
const changeHero=()=>{
  hero.value=`大方`
}
const changePrice=()=>{
  price.value++
}
</script>

<template>
  <button @click="changeHero">hero</button>
  <button @click="changePrice">price</button>
</template>

由此可见，如果是监听多个数据，那么其中一个发生改变，newValue和oldValue拿到的整个数组

5.父传子 & 子传父 ​

​ 在vue3中，由于我们的代码都是卸载setup中，所以我们无法像vue2一样直接使用props:进行组件通信，因此vue3为我们提供了编译器宏函数:defineProps()。

【Ⅰ】父传子 ​

vue
//父
<script setup>
import Son from '@/components/SonBase.vue'
import {ref} from 'vue'
const salary = ref(1600);

</script>

<template>
  <Son :salary="salary"></Son>
</template>
vue
//子
<script setup>
    // 由于写了setup，所以无法直接配置prop选项
    // 所以:我们需要借助 编译器宏 函数接受父组件传递的数据,同时提供一个变量接受返回值
const props = defineProps({
    salary:Number
})
	// 在脚本中想要获取父组件传来的值需要加上 变量.值 
console.log(props.salary)
</script>

<template>
	// 在模板中可以直接使用
    <div>Son:{{ salary }}</div>
</template>

<style scoped>
div{
    width: 300px;
    height: 150px;
    text-align: center;
}
</style>

【Ⅱ】子传父 ​

​

​ 在vue2中我们想要实现子传父需要使用this.$emit来进行通知父组件，但到了vue3中,我们同样无法直接使用$emit，因此我们也同样需要借助vue3提供的 编译器宏 函数 defineEmits()。

vue
//父
<script setup>
import Son from '@/components/SonBase.vue'
import {ref} from 'vue'
const salary = ref(1600);

const setSalary = (num)=>{
  salary.value=num
}
</script>

<template>
  <Son
    @changeSalary="setSalary " 
    :salary="salary">
  </Son>
</template>
vue
//子
<script setup>
// defineEmits()中需要提供一个数组，数组中包含我们要通知父组件的方法字符串。
// 返回值用一个变量接收
const emit=defineEmits(['changeSalary'])
const props = defineProps({
    salary:Number
})

const addSalary = () =>{
    //变量名(`方法名`,参数)
    emit(`changeSalary`,props.salary+100)
}
console.log(props.salary)

</script>

<template>
    <div>Son:{{ salary }}</div>
    <button @click="addSalary">增加</button>
</template> 

<style scoped>
div{
    width: 300px;
    height: 150px;
    text-align: center;
}
</style>

7.provide & inject ​

​

​ 在vue2中我们也是使用过provide&inject，但是vue2的基本上没怎么用，而在vue3中，我们会常常使用provide&inject进行跨组件通信。

vue
//	顶层组件
<script setup>
import centerBase from '@/components/CenterBase.vue'
    // 1. 导入provide
import { provide,ref } from 'vue';
const count =ref(1600)
 // 2. 通过provide(`键`,值)进行传递
provide(`count`,count)
 // 3.由于子组件无法直接修改值，所以也能传个方法过去修改值   
provide(`changeCount`,(num) =>{
  count.value=num
})
</script>

<template>
  <div>
    <h3>外部组件</h3>
    <centerBase></centerBase>
  </div>

</template>

<style scoped>
div{
  width: 500px;
  height: 450px;
}
</style>
vue
//	中间组件
<script setup>
import bottonBase from '@/components/BottonBase.vue'
</script>

<template>
<div>
    <h3>中间组件</h3>
    <bottonBase></bottonBase>
</div>
</template>

<style scoped>
div{
    width: 300px;
    height: 250px;
}
</style>
vue
//	底层组件
<script setup>
    // 1. 导入inject
import { inject } from 'vue';

	// 同理方法也能

    changeCount(3600)

8.defineOptions ​

​

​ 我们在前面用vue3的时候可能有点别扭，因为我们的< script>都是用setup修饰的，这就导致我们的代码都是写在setup函数中的，也就说我们无法使用其他的平级函数，即使vue3给了我们一些编译宏可以让我们使用生命周期函数与父子通信，但还是有一些没有解决，

​ 比如如果我们的组件名是单个单词的话，编译器是会报错的，而我们为了解决报错就需要添加name这一配置项，但又因为script中全是setup的作用域，就导致有些时候会出现两个script，一个带setup，一个不带，而像name这种就会写在不带的中。

​ 而这别扭的现象在vue3.3后得到了修复。vue3.3新推出了一个新的编译宏，defineOptions,主要用来定义options API选项，我们可以用defineOptions来定义任意的选项，当然props、emit、expose、slots除外（因为这些都出了单独的编译宏）。

vue
<script setup>
import { inject } from 'vue';

const count =inject(`count`)
const changeCount = inject(`changeCount`)
const setCount=()=>{
    changeCount(3600)
}

// defineOptions({})中写即可
defineOptions({
    name:`ButtonOne`
})
</script>

4.storeToRefs ​

​ 前面我们也提到过，我们不能直接对store对象进行解构，这样会丢失数据的响应性，这是因为store本质上是一个用reactive包裹的对象，而reactive本质上式监听整个对象，而我们直接进行解构就相当于定义了一个变量，变量的值式store上的一个值。

​ 如果我们真的需要进行解构获得变量并且不丢失响应式，就可以使用storeToRefs。

vue
<script setup>
import {storeToRefs} from 'pinia' //导入storeToRefs
import { useHero } from './stores/hero'; 

const hero =useHero() 
const {salary} =storeToRefs(hero) //storeToRefs(store对象)
</script>

<template>
  <h3>{{ salary }}</h3>//可正常使用，且是响应式
  <h3>父组件---{{ hero.name }}--{{ hero.salary }} <button @click="hero.addSalary">增加</button></h3>
</template>

5.FormData ​

​

​ FormData类型的对象，在个人看来就是键值对，我们可以看下mdn对于该对象的描述。

FormData 接口提供了一种表示表单数据的键值对 key/value 的构造方式，并且可以轻松的将数据通过XMLHttpRequest.send() 方法发送出去，本接口和此方法都相当简单直接。如果送出时的编码类型被设为 "multipart/form-data"，它会使用和表单一样的格式。

如果你想构建一个简单的GET请求，并且通过form的形式带有查询参数，可以将它直接传递给URLSearchParams。

​ 而我根据网上查找的资料，基本上可以得出一个结论，如果是传输的是表单数据的话，使用FormData，其余用json。或许在未来我会对FormData有更深刻的理解，到时这里的笔记可能就不会适用了。

7.axios.patch ​

​ patch方法是用来更新局部数据的，比如说我们有一张user表，但我们这次请求只需要更改user中的密码，那么我们就可以使用patch来局部更新，因为put是要传整个表的数据过去进行完整更新。具体可以看下博客文章。

patch方法用来更新局部资源，这句话我们该如何理解？

假设我们有一个UserInfo，里面有userId， userName， userGender等10个字段。可你的编辑功能因为需求，在某个特别的页面里只能修改userName，这时候的更新怎么做？

人们通常(为徒省事)把一个包含了修改后userName的完整userInfo对象传给后端，做完整更新。但仔细想想，这种做法感觉有点二，而且真心浪费带宽(纯技术上讲，你不关心带宽那是你土豪)。

于是patch诞生，只传一个userName到指定资源去，表示该请求是一个局部更新，后端仅更新接收到的字段。

而put虽然也是更新资源，但要求前端提供的一定是一个完整的资源对象，理论上说，如果你用了put，但却没有提供完整的UserInfo，那么缺了的那些字段应该被清空PATCH 与 PUT 属性上的一个重要区别还在于：PUT 是幂等的，而 PATCH 不是幂等的。 幂等是一个数学和计算机学概念，在计算机范畴内表示一个操作执行任意次对系统的影响跟一次是相同。

如：POST 方法不是幂等的，若反复执行多次对应的每一次都会创建一个新资源。如果请求超时，则需要回答这一问题：资源是否已经在服务端创建了？能否再重试一次或检查资源列表？而对于幂等方法不存在这一个问题，我们可以放心地多次请求。

补充:

最后再补充一句，restful只是标准，标准的意思是如果在大家都依此行事的话，沟通成本会很低，开发效率就高。但并非强制(也没人强制得了)，所以你说在你的程序里把方法名从put改成patch没有任何影响，那是自然，因为你的后端程序并没有按照标准对两个方法做不同处理，她的表现自然是一样的

8.解决跨域问题 ​

​ 我们如果想要在vue3中解决跨域问题就可以进行以下配置

// request.js 
// 该js的主要用途是与原axios分开，防止污染，同时对新的实例进行统一配置
// 设置及基地址为 /api ,当然值不是这个也行，该值只是为了正则查询罢了
const baseUrl = '/api'

const instance = axios.create({
  baseURL: baseUrl,
  timeout: 1000
})

​ 设置好了axios的baseUrl，我们就可以去vite.config.js中配置一下

js
// vite.config.js
export default defineConfig({
 ...
  server:{
      // 该端口为我们vue调用的是哪个端口，如果改成了8081,那么vue的url也将变为localhost:8081/
    port:'5173',
    proxy: {
        // 这个就类似于nginx的location /api{},当我们的请求是/api开头就会被接收到，
        // target 设置我们的目标url
        // changeOrigin 为是否允许跨域
        // rewrite 我们通过rewrite 进行对url的切换，
      	# 假设我们发了个http://localhost:5173/api/user/select
        # 就会被该location拦截到，并且通过替换将/api替换成''
        # 再通过与target拼接
        # 最后就变成了"http://localhost:8080/user/select"
      '/api' :{
        target:"http://localhost:8080",
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/api/, '') 
      }
    }
    
  }
})

9.解决修改文件夹名或者移动文件夹后pnpm dev not found model问题 ​

​ 根据我个人上网搜索查询得到的资料，以及个人猜测，node_models中可能配了一些跟路径有关的东西，而且路径可能还是绝对路径，这样就能解释为什么修改文件夹名字或者移动文件夹后会报 not found model 了。

10.Promise函数的用法 ​

​ Promise函数本身就是一个异步请求的容器，说白了就是我们在里面发送异步请求，然后根据里面的条件再来判断是否成功，也不能这么说，直接看代码会清楚一些。

js
// 这里我们可以看到getList()返回一个promise对象
// 在刚开始创建的时候,promise实例的状态是pending(待定的)
const getList =()=>{
    // promise的构造方法接收一个函数
    // 函数的第一个参数是resolve,第二个参数是reject。这两个参数都是script提供
    
    // reject调用后会将promised的状态设置为rejected(rejected：拒绝,这里引申为失败)
    return new Promise((resolve,reject)=>{
        // 通过axios请求向后端请求数据
        userSelectService().then(res=>{
            // if判断，注意往resolve和reject传的值
            if(res.code ===1 && res.list){
                // resolve调用后会将promised的状态设置为fullfiled(fullfile：完成，这里引申为成功)
                resolve(res)
            }else{
                reject(res)
            }
        })
    })
}
// Promise对象的then()接收一个函数
// 同样的，函数也是接收两个回调，第一个是成功时回调，第二个是失败时回调
getList().then(data=>{
    // 该data是上面resolve(res)中的res，也就是说上面存的值我们可以在这里取出
    console.log(data)
},data=>{
    // 同样的，这里的data是reject(res)中的res
    console.log(data)
})

--------------------------------------------------------------------------
// 同时，我们要知道resolve()中传的值如果是一个promise对象，那么resovle()将会原封不动的返回这个对象
// 那么我们可以利用这个特性完成链式操作

function runAsync1() {
    const p = new Promise(function(resolve, reject) {
        setTimeout(function(){
            console.log('执行完成1');
            resolve(1);
        }, 2000)
    })
    return p;    
}

function runAsync2() {
    const p = new Promise(function(resolve, reject) {
        setTimeout(function(){
            console.log('执行完成2');
            resolve(2);
        }, 2000)
    })
    return p;    
}


function runAsync3() {
    const p = new Promise(function(resolve, reject) {
        setTimeout(function(){
            console.log('执行完成3');
            resolve(3);
        }, 2000)
    })
    return p;    
}

runAsync1().then(function(data) {
    console.log(data);
    return runAsync2();
}).then(function(data) {
    console.log(data);
    return runAsync3();
}).then(function(data) {
    console.log(data);
    return '直接返回数据'
}).then(function(data) {
    console.log(data);
})
