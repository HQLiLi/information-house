Epubjs使用文档 ​

前言：由于该库在网络上的资源属实较少，且没找到相关的视频，因此本部分都是本人通过网络及个人摸索得出的一些使用方法，若未来有机会再次开发相关内容的项目时，本文档可立大功。

在实际使用之前，我们要知道Epubjs的原理，本质上是创建一个ifame标签【可以想成iframe中就是一个网页】，然后里面就是epub里面的内容。

​ 我们需要先了解一下.epub。epub文件本质上就是个压缩包，跟.jar一样，我们可以把后缀改为.zip，解压后我们就能得到里面的内容。

1.创建epub对象 ​

​ 创建epub对象的方法大致分为两种，一种为本地资源创建，一种为网络资源创建，

js
const book = ref(null)
// 方法：Epub(target, options)
// -本地资源的创建很简单，直接把资源路径往上一贴就行
book.value = epub('XX.epub')
// -网络资源的话，文档其实是留了坑的，因为网络资源并不能直接创建，它是有一定的规则的
// 1.如果是直接通过链接来返回的话，则链接必须以.epub结尾【关于这点我还没有研究清楚，因此更推荐下面的方式】
// 2.通过Base64,使用该类型需要设置options.encoding='base64'【这种方法是最简便的】
book.value = epub(res.data, { encoding: 'base64' })

2.创建【预】渲染对象 ​

​ 在epub对象创建成功后，我们就要开始创建epub的渲染对象

js
// epub对象
const book = ref(null)
// 该为渲染对象
const red = ref(null)

// 方法renderTo(el, options)
// el为我们需要渲染的dom元素的id 如：<div id="book"></div>
red.value = book.value.renderTo('book', {
    // 该选项有两个值:'continuous' | 'default'
    // 如果是continuous的话，就会预加载多个view视图【就是页】
    // 如果是defalut的话，只会加载一个view视图
    manager: 'continuous', 
    // 是否支持翻页,该属性是在pe上使用的拖动翻页使用,网页版用不着
    snap: true,
    // 设置大小,还有一个width属性
    height: '85vh',
    // iframe 沙盒是否能够执行js脚本【该属性必须为true,不然会渲染失败，这是因为安全机制更新了】
    allowScriptedContent: true,
    // 阅读方式:'paginated' | 'scrolled',前者为双页 左右翻，后者为单页 上下翻
    flow: 'paginated',
    // 忘记了
    boxSizing: 'border-box'
  })

3.正式渲染EPub ​

​ 在前面我们已经实现了epub的预渲染，但我们还是看不到内容，这是因为这本书还是合上的，我们需要手动打开。

js
// 渲染对象
// display() 打开epub
red.value.display()

4.获取目录 ​

​ 我们如果想要获得目录，就需要在epub对象预设定时获得【渲染和预设定不是一个东西，渲染是显示这玩意长啥样，预设定是这玩意有啥功能】

js
// 目录对象
const cata = ref(null)

// 预设定
book.value.ready
.then(() => {
  // 获取目录
  // 注:book.navigation.toc必须在ready中获取，否则会拿不到
  cata.value = book.value.navigation.toc
  console.log(cata.value)
})
.then(() => {
  // 设置分页【该内容后面会讲，这里先】
  location.value = book.value.locations.generate(
    750 * (window.innerWidth / 375)
  )
})

​ 我们通过控制台打印可得知目录其实就是数组，里面存了一堆对象，而我们要制作目录就是使用对象里面的lable和href，至于如何用href来实现跳转，后面会说.

5.跳转 ​

​ 我们在前面已经知道可以在目录中拿到href，其实不仅仅能通过href来跳转，还能用toc来【后面会说】。

js
//跳转
const catalogueSkip = (href) => {
  // 由于每本epub的目录都不一定一致，但是整体的文件目录格式应该一致的，所以我们要进行判断。
  // 若目录给的href并没有前面的Text/ 的话我们需要手动加，因为要实现跳转必须是完整目录【默认根路径是从OEBPS开始】
    if (href.includes('Text')) {
        // 同样是使用display()方法,我们可以理解将书翻到指定位置
    red.value.display(href)
  } else {
    red.value.display(`Text/${href}`)
  }
}

6.前进&后退 ​

js
// Epubjs给我们提供了两个方法来完成向前向后，方法在渲染对象上

// 前进
const go = () => {
  red.value.next()
}

// 后退
const backward = () => {
  red.value.prev()
}

7.分页【实现进度条及进度条拖拽的前提】 ​

​ 虽然我们在渲染时就能设置了分页显示，但是epub本身是并没有分页的，我们还是要手动分【这里很逆天，渲染对象的分页显示和容易跟分页功能搞混引起误解】

js
// 同样，分页功能也是在预设定时进行设置，且只能在这设置
// locations对象是定位对象

// 预设定
book.value.ready
.then(() => {
  // 获取目录
  // 注:book.navigation.toc必须在ready中获取，否则会拿不到
  cata.value = book.value.navigation.toc
  console.log(cata.value)
})
.then(() => {
  // 设置分页
  location.value = book.value.locations.generate(
    750 * (window.innerWidth / 375)
  )
})

8.进度条 ​

​ 在实现进度条前我们必须要完成7.分页，否则将无法实现该功能

vue

<script setup>
    ...
// 进度条
const slider = ref()
// 进度指数
const range = ref(0)
// 进度跳转 【该方法是如果range改变就会触发】
const changeRange = () => {
  let percent = range.value / 100
  location.value =
    percent > 0 ? book.value.locations.cfiFromPercentage(percent) : 0
    // 如果没有进行分页的话，就会拿到-1
    // 如果分页的话，就会拿到这样的字符串：epubcfi(/6/36!/4,/1426/1:5,/1536/1:49)
    // 我们可以将cfi理解为页码，我们通过页码进行跳转
  console.log(location.value)
    // 页码跳转
  red.value.display(location.value)
}
const inputRange = () => {
  let percent = range.value / 100
  location.value =
    percent > 0 ? book.value.locations.cfiFromPercentage(percent) : 0
}

// 设置进度条位置，即手动翻页进度条跟着动
const drag = (cfi) => {
  if (cfi !== null && cfi !== '') {
    range.value = book.value.locations.percentageFromCfi(cfi) * 100
  }
}
</script>


<template>
...
        <!-- 进度条 -->
        <!-- 该处用了element-plus的进度条组件，本质上就是type为range的input -->
        <el-slider
          v-model="range"
          style="width: 50vh"
          class="range"
          @change="changeRange"
          @input="inputRange"
          ref="slider"
        />
</template>

9.实现左翻右翻【渲染对象.on()】 ​

js
// 我们可以通过渲染对象.on()方法来监听事件
// 可以监听的事件大致有"keydown", "keyup", "keypressed", "mouseup", "mousedown", "click", "touchend", "touchstart", "touchmove"【没有鼠标滚轮】

  // 左翻右翻
  // 该方法是通过我们点击屏幕来根据点击的位置来判定是左翻还是右翻
  red.value.on('click', (event) => {
    const leftInstance = event.screenX
    const screenWidth = window.innerWidth
    const cfi = red.value.currentLocation().end.cfi
    if (leftInstance < screenWidth / 2) {
      drag(cfi)
      backward()
    } else {
      drag(cfi)
      go()
    }
  })
